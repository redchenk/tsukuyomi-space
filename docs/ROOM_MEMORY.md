# Room 长期记忆

Room 的长期记忆是一个浏览器侧外挂模块，不依赖后端 API，也不会改变站点核心运行路径。

## 存储位置

- 记忆保存在浏览器 IndexedDB：`tsukuyomi-room-memory`。
- 每条记忆按 `userKey` 分桶：
  - 已登录用户：`user:<user.id 或 username/email>`。
  - 未登录访客：`guest:<本地随机 id>`。
- 不会上传到服务器，也不会与其他用户共享。

## 向量检索

当前实现使用本地轻量哈希向量：

- 把对话文本切成词元和相邻片段。
- 映射为固定维度向量。
- 用户新消息到来时，只在当前 `userKey` 的记忆里计算相似度。
- 取最相关的少量记忆注入 Room LLM 的 system prompt。

这不是外部 embedding 服务，因此不会泄露对话内容。后续如果引入真正的 embedding API，必须继续保持“用户侧请求”和“按用户隔离”的原则。

## 写入策略

每次 Room 对话完成后，会保存：

- 访客身份名。
- 用户本轮消息。
- 八千代本轮回复。
- 本地向量。
- 创建时间。

单个用户最多保留 240 条，超过后自动删除最旧记忆。

## 用户控制

Room 设置面板提供：

- 开关本地记忆网络。
- 查看当前身份和记忆数量。
- 清空当前用户的本地记忆。

关闭记忆后，不再检索或写入；已有数据会保留，直到用户手动清空。
